# ADR-051: Cross-viewer vector consistency CI

## Status
Proposed (Phase 9)

## Context
SVG and PDF rendered by different viewers (Chrome, Firefox,
Inkscape, Adobe Reader) can look subtly different due to text
kerning, anti-aliasing, and path rendering differences. Phase 9
must verify that exported figures look consistent across viewers.

## Decision
New CI job (Ubuntu-only) that renders canonical test plots and
compares renderings via PSNR.

### Pipeline
1. Export each canonical plot to SVG and PDF.
2. Render SVG via: Playwright headless Chrome, Playwright
   headless Firefox, Inkscape CLI (`inkscape --export-type=png`).
3. Render PDF via: pdftocairo (`pdftocairo -png`).
4. Compute PSNR between each pair of renderings.
5. Threshold: PSNR > 40 dB.

### Canonical test plots (git-committed references)
- Simple line plot (3 series)
- Scatter with 6 marker shapes
- Bar chart
- Heatmap with Viridis colormap
- Plot with LaTeX axis labels (added after M9.4)
- Plot with annotations (added after M9.5)
- Each plot with each built-in font

References stored in tests/fixtures/vector-consistency/ as PNGs.
Regenerated by developer script when rendering pipeline changes.

### Rollout
- M9.3: job runs, warn-only (does not block merge)
- M9.4+: job blocks merge

### Tools (all apt-installable)
- Playwright: `npx playwright install --with-deps chromium firefox`
- Inkscape: `apt install inkscape`
- pdftocairo: `apt install poppler-utils`
- PSNR: computed via Python Pillow + numpy (3-line script)

macOS CI excluded (renderer-specific, not platform-specific).

## Consequences
- + Automated consistency verification
- + Catches regressions in SVG/PDF generation
- + PSNR is simple to compute, threshold, and debug
- + Warn-only rollout prevents premature CI breakage
- - CI job adds ~3 minutes (rendering + comparison)
- - Playwright install adds ~200MB to CI cache
- - Threshold tuning may be needed as rendering pipeline evolves
- - macOS not covered (acceptable: renderer behavior is
  OS-independent for headless Chrome/Firefox)

## Alternatives considered
- **Spec compliance only** (validate SVG XML, PDF structure):
  Rejected. Viewers routinely ignore parts of the spec. A
  spec-valid SVG can look different in Chrome vs Inkscape.
- **Manual visual testing**: Rejected. Not scalable, not
  reproducible, blocks on human availability.
- **SSIM instead of PSNR**: Rejected. SSIM is harder to
  threshold (structural vs pixel difference), and for our use
  case (mostly identical images with minor text differences)
  PSNR is more intuitive and debuggable.
- **Per-pixel exact match**: Rejected. Anti-aliasing differences
  make exact match impossible across renderers.
